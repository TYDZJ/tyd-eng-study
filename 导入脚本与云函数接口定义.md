# 导入脚本与云函数接口定义（V1）

## 1. 文档目标

本文件用于直接指导编码，覆盖两部分：

- 词库导入脚本（仅脚本，无管理后台）
- 用户端核心云函数接口（学习/复习闭环）

与 `后端开发文档.md` 保持一致，已固定参数：

- 词书：`CET-4`、`CET-6`
- 每日新词默认值：`20`
- 每次 session 默认值：`10`（可选 `5~40`，步长 `5`）
- 复习策略：待复习无上限累积，不重复新增
- 复习算法：`SM2 简化版`
- 登录策略：必须登录，不支持游客
- 音频：V1 不支持

---

## 2. 导入脚本设计

## 2.0 仓库内已实现的第一版（小样本默认 50 条/书）

项目已包含可运行脚本（`scripts/import/`）：

| 命令 | 说明 |
|------|------|
| `npm run import:prepare` | 从 `english-vocabulary-master` 读取 `json-sentence/CET4_1.json`、`CET6_1.json`，每本各取前 **50** 条，生成 `scripts/import/data/*.json` |
| `npm run import:verify` | 校验上述 JSON 的关联与计数 |
| `npm run import:push` | 支持两种方式写入云库 `books` / `words` / `book_words`：① 配置 `TCB_SECRET_ID` + `TCB_SECRET_KEY`；② 先执行 `npx cloudbase login`，走 CloudBase CLI 登录态（无 API 密钥也可） |

环境变量：

- `ENGLISH_VOCAB_ROOT`：词库根目录，默认与仓库同级的 `../english-vocabulary-master`
- `IMPORT_WORD_LIMIT`：每本书截取条数，默认 `50`
- `TCB_ENV_ID`：云环境 ID；不设则从 `src/config/wx-cloud.js` 读取 `WX_CLOUD_ENV_ID`

若使用 CloudBase CLI 登录态，请先执行：

- `npx cloudbase login`
- `npx cloudbase env list --env-id <你的环境ID>`

生成的 `book_words` 使用稳定 `_id`：`{book_id}_{seq}`（如 `cet4_1`），便于重复执行 `set` 覆盖。

## 2.4 字段规范化规则

- `word` 统一 `trim().toLowerCase()`
- `word_id = word`
- 去重规则：
  - `words` 按 `word_id` 去重
  - `phrases` 按 `phrase + translation` 去重
  - `sentences` 按 `sentence + translation` 去重
- 空值处理：
  - `uk/us` 为空则写空字符串
  - `translations/phrases/sentences` 无值写空数组
- 非法词条过滤：
  - 空 `word`
  - `translations` 为空且 `phrases/sentences` 也为空（可选过滤）

---

## 2.5 导入执行顺序

固定顺序，避免关联失败：

1. `import-books.js`
2. `import-words.js`
3. `import-book-words.js`
4. `verify-import.js`

分批策略（微信云数据库）：

- `words`：每批 `200~500`
- `book_words`：每批 `500~1000`
- 记录断点（最后成功批次），支持重试

---

## 2.6 导入校验清单

`verify-import.js` 至少检查：

1. `books` 中存在 `cet4`、`cet6`
2. `book_words` 中每个 `word_id` 在 `words` 可命中
3. `cet4/cet6` 的 `word_count` 与映射数量一致
4. 随机抽样 20 个词，检查 `translations` 不为空

## 4. 通用接口约定

## 4.1 鉴权

- 所有用户端函数通过 `wxContext.OPENID` 识别用户
- 若拿不到 `OPENID`，返回未登录错误
- 不允许传入他人 `openid`

## 4.2 响应结构

统一返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

错误示例：

```json
{
  "code": 1001,
  "message": "UNAUTHORIZED",
  "data": null
}
```

建议错误码：

- `0`：成功
- `1001`：未登录/鉴权失败
- `1002`：参数错误
- `1003`：资源不存在
- `2001`：状态冲突（session 已结束等）
- `9001`：服务异常

## 5. 接口定义

## 5.1 `getBookList`

用途：获取词书列表 + 用户进度摘要

入参：无

返回：

```json
{
  "books": [
    {
      "book_id": "cet4",
      "name": "大学英语四级",
      "word_count": 4500,
      "learned_count": 320,
      "mastered_count": 80,
      "review_due_count": 56,
      "current_seq": 321
    }
  ]
}
```

---

## 5.2 `getBookDetail`

用途：获取某词书详细进度

入参：

```json
{
  "book_id": "cet4"
}
```

返回：

```json
{
  "book_id": "cet4",
  "name": "大学英语四级",
  "word_count": 4500,
  "learned_count": 320,
  "mastered_count": 80,
  "review_due_count": 56,
  "today_new_quota": 20,
  "session_default_size": 10,
  "session_size_options": [5, 10, 15, 20, 25, 30, 35, 40]
}
```

---

## 5.3 `getTodayPlan`

用途：返回今日待学/待复习统计

入参：

```json
{
  "book_id": "cet4"
}
```

返回：

```json
{
  "book_id": "cet4",
  "today_new_quota": 20,
  "today_new_remaining": 12,
  "review_due_count": 56,
  "review_overdue_count": 13
}
```

说明：

- `review_due_count` 无上限，历史未复习完持续累积
- 同一词只保留一条状态，不会重复新增

---

## 5.4 `startSession`

用途：开始一次学习/复习 session，返回本次词列表

入参：

```json
{
  "book_id": "cet4",
  "mode": "mixed",
  "session_size": 10
}
```

参数规则：

- `mode`：`learn|review|mixed`
- `session_size`：可选，默认 `10`，取值必须属于 `[5,10,15,20,25,30,35,40]`

返回：

```json
{
  "session_id": "sess_xxx",
  "mode": "mixed",
  "target_count": 10,
  "words": [
    {
      "word_id": "talk",
      "word": "talk",
      "uk": "tɔːk",
      "us": "tɔk",
      "translations": [{ "type": "v", "translation": "说话；谈话" }],
      "phrases": [],
      "sentences": []
    }
  ]
}
```

选词规则（`mixed`）：

1. 先取 `next_review_at <= now` 的复习词（按逾期优先）
2. 不足部分补新词（按 `book_words.seq` 从 `current_seq+1` 开始）
3. 总数凑到 `session_size`

---

## 5.5 `submitWordResult`

用途：提交当前词反馈，更新调度并返回下一词信息

入参：

```json
{
  "session_id": "sess_xxx",
  "book_id": "cet4",
  "word_id": "talk",
  "rating": "good",
  "cost_ms": 4800
}
```

参数规则：

- `rating` 仅允许：`again|hard|good|easy`

返回：

```json
{
  "session_id": "sess_xxx",
  "finished_count": 3,
  "target_count": 10,
  "next_review_at": "2026-04-12T09:00:00.000Z",
  "next_word_id": "ability",
  "session_finished": false
}
```

当 session 结束时：

```json
{
  "session_id": "sess_xxx",
  "finished_count": 10,
  "target_count": 10,
  "session_finished": true
}
```

## 6. SM2 简化版落地规则（V1）

## 6.1 状态字段

`user_word_state` 使用字段：

- `reps`
- `ease_factor`（初始 2.5）
- `interval_days`
- `last_review_at`
- `next_review_at`
- `lapses`
- `status`

## 6.2 rating 到质量分映射

- `again -> 0`
- `hard -> 3`
- `good -> 4`
- `easy -> 5`

## 6.3 更新规则（简化）

1. 若 `again`：
   - `reps = 0`
   - `interval_days = 1`
   - `lapses += 1`
   - `status = learning`

2. 否则：
   - `reps += 1`
   - `ease_factor = max(1.3, ease_factor + (0.1 - (5-q)*(0.08 + (5-q)*0.02)))`
   - `interval_days`：
     - 若 `reps == 1`：`1`
     - 若 `reps == 2`：`3`
     - 否则：`round(interval_days * ease_factor)`
   - `status`：
     - `good/easy` 且 `reps >= 3` -> `reviewing`
     - `reps >= 8` 可标记 `mastered`（可选）

3. `next_review_at = now + interval_days`

## 9. 验收用例（最小）

1. 用户首次进入 `cet4`，能开始 10 词 session
2. 提交 10 次 `submitWordResult` 后 session 正常结束
3. 次日 `getTodayPlan` 中出现待复习词
4. 当天未复习完，次日待复习数量累积且不重复
5. 用户切换到 `cet6`，进度互不影响

达到以上 5 条，即可视为后端 MVP 可用。
