# 英语单词小程序后端开发文档（微信云开发）

## 1. 目标与范围

本项目基于微信云开发（CloudBase）实现类似“不背单词”的学习/复习流程，核心目标：

- 词库管理：支持词书（四级、六级等）及其词条数据管理
- 学习进度：记录用户在某词书的学习进度、掌握情况
- 每日任务：生成“今日新词 + 今日复习词”清单
- 学习流程：按批次（每次 N 个词）推进学习与复习

数据源使用开源词库：`english-vocabulary`（含 `json-full/json-simple/json-sentence` 三类格式）。

### 1.1 当前已确认需求（2026-04-10）

- 首发词书范围：仅 `CET-4`、`CET-6`
- 学习策略参数：每次学习词数支持 `5~40`，步长 `5`
- 每日新词默认值：`20`
- 每日复习策略：**无上限累积**；历史未复习完的单词持续保留在待复习池，且同一单词不会重复新增
- 每次 session 默认值：`10`（用户可改为 `5~40`，步长 `5`）
- 评分档位：采用 `again / hard / good / easy` 四档
- 复习算法：V1 采用 `SM2 简化版`（优先快速开发）
- 音频能力：V1 不支持发音音频
- 登录策略：不支持游客学习，必须登录
- 管理后台：仅脚本导入，不做 Web 管理端
- 规模预估：个人毕设，用户数不超过 10 人

## 2. 词库数据结构分析与选型

你给的示例里：

- `sample_json_simple`：`word + translations + phrases`
- `sample_json_sentence`：在 simple 基础上增加 `us/uk + sentences`
- `sample_json_full`：原始层级深，信息最全（同近义词、词根等），字段嵌套复杂

建议：

- **线上主词库使用 `json-sentence` 作为标准结构**（信息完整且结构平衡）
- `json-full` 作为离线增强数据来源，后续需要时再补充（如同义词、词根）
- 前期不要把 full 的所有字段直接入库，避免文档过大、查询慢、开发复杂

## 3. 云数据库集合设计

> 命名建议使用 snake_case，方便和云函数字段保持一致。

### 3.1 `books`（词书主表）

用途：词书元信息（四级、六级、考研...）

核心字段：

- `_id`: string（建议 `cet4`, `cet6`, `kaoyan` 这种业务 id）
- `name`: string（词书名）
- `level`: number（难度等级，可选）
- `word_count`: number（总词数）
- `description`: string（可选）
- `cover_url`: string（可选）
- `source`: string（数据来源版本）
- `created_at` / `updated_at`: serverDate

索引建议：

- `name` 普通索引

---

### 3.2 `book_words`（词书-单词映射表）

用途：某词书包含哪些词，及顺序（学习顺序非常关键）

核心字段：

- `_id`: string（自动）
- `book_id`: string（关联 `books._id`）
- `word_id`: string（关联 `words._id`）
- `seq`: number（词书内顺序，1..N）
- `tags`: string[]（可选，如核心词/高频词）

索引建议（重点）：

- 复合索引：`book_id + seq`
- 复合索引：`book_id + word_id`（唯一约束逻辑由代码保证）

---

### 3.3 `words`（单词主表）

用途：去重后的词条数据（跨词书可复用）

核心字段（推荐与 `json-sentence` 对齐）：

- `_id`: string（建议使用规范化 word，如 `talk`）
- `word`: string
- `uk`: string（可空）
- `us`: string（可空）
- `translations`: array<{ type: string, translation: string }>
- `phrases`: array<{ phrase: string, translation: string }>
- `sentences`: array<{ sentence: string, translation: string }>
- `extra`: object（后续放同义词/词根等扩展）
- `created_at` / `updated_at`

索引建议：

- `word` 唯一索引（或直接作为 `_id`）

---

### 3.4 `user_book_progress`（用户词书学习总进度）

用途：展示“某词书学了多少”

核心字段：

- `_id`: string
- `openid`: string
- `book_id`: string
- `current_seq`: number（当前学习游标）
- `learned_count`: number（已学习）
- `mastered_count`: number（已掌握）
- `review_due_count`: number（待复习数，冗余字段可异步更新）
- `last_learn_date`: string（YYYY-MM-DD）
- `created_at` / `updated_at`

索引建议：

- 复合索引：`openid + book_id`（唯一）

---

### 3.5 `user_word_state`（用户-单词状态表，核心）

用途：驱动学习/复习调度，记录每个词的掌握轨迹

核心字段：

- `_id`: string
- `openid`: string
- `book_id`: string
- `word_id`: string
- `status`: string（`new|learning|reviewing|mastered`）
- `fsrs_state`: object（或 sm2_state，保存记忆算法参数）
- `ease_factor`: number（如使用 SM2）
- `interval_days`: number
- `reps`: number（连续正确次数）
- `lapses`: number（遗忘次数）
- `last_result`: string（`again|hard|good|easy`）
- `last_review_at`: datetime
- `next_review_at`: datetime（关键调度字段）
- `created_at` / `updated_at`

索引建议（重点）：

- 复合索引：`openid + book_id + next_review_at`
- 复合索引：`openid + book_id + status`
- 复合索引：`openid + book_id + word_id`（唯一）

---

### 3.6 `study_sessions`（学习会话表）

用途：每次进入学习/复习生成一个 session，便于追踪和统计

核心字段：

- `_id`: string
- `openid`: string
- `book_id`: string
- `mode`: string（`learn|review|mixed`）
- `target_count`: number（本次目标词数）
- `word_ids`: string[]（本次词列表快照）
- `finished_count`: number
- `status`: string（`running|finished|aborted`）
- `started_at` / `finished_at`

索引建议：

- `openid + book_id + started_at`

---

### 3.7 `study_logs`（做题/记忆日志）

用途：行为明细，用于分析与后续调参

核心字段：

- `_id`
- `session_id`
- `openid`
- `book_id`
- `word_id`
- `action`: string（`learn_show|review_result|audio_play|example_expand`）
- `rating`: string（`again|hard|good|easy`，可空）
- `cost_ms`: number
- `created_at`

索引建议：

- `session_id`
- `openid + created_at`

---

## 4. 学习与复习流程（后端视角）

### 4.1 今日任务生成

输入：`openid + book_id + 当日目标新词数 new_quota + 单次学习词数 batch_size`

步骤：

1. 查 `user_word_state` 中 `next_review_at <= now` 的词，得到待复习池
2. 按优先级（逾期时间、遗忘次数）选出今日复习词
3. 若新词额度未满，从 `book_words` 按 `current_seq` 向后取新词
4. 写入/更新 `study_sessions`，返回本次列表

### 4.2 学习卡片提交

用户对某词选择 `again/hard/good/easy` 后：

1. 根据记忆算法更新 `user_word_state`
2. 计算新的 `next_review_at`
3. 写 `study_logs`
4. 返回下一个词（或 session 完成）

### 4.3 推荐记忆算法

算法说明：

- **SM2 简化版**：实现复杂度低，适合快速上线（你的场景建议选它）
- **FSRS**：效果更好但实现和调参更复杂，适合后续优化

本项目决策：

- **V1 固定使用 SM2 简化版**
- 在 `user_word_state` 中预留 `fsrs_state` 字段，后续可无痛升级到 FSRS

## 5. 云函数（后端 API）设计

建议拆分为 3 组云函数：

### 5.1 词库函数（管理端）

- `adminImportBooks`：导入词书元数据
- `adminImportWords`：导入单词（支持分批）
- `adminBindBookWords`：写入词书-词映射

> 管理端函数必须做鉴权（白名单 openid 或管理员角色）。

### 5.2 学习函数（用户端核心）

- `getBookList`：获取词书列表 + 用户进度摘要
- `getBookDetail`：词书详情（总词数、已学、待复习）
- `startSession`：开始学习/复习，返回本次词列表
- `submitWordResult`：提交单词结果并返回新调度
- `getTodayPlan`：获取今日待学/待复习统计

### 5.3 同步/统计函数

- `refreshBookProgress`：重算 `user_book_progress`
- `dailyAggregate`：生成日报统计（可定时触发）

## 6. 数据导入方案（推荐落地步骤）

## 6.1 预处理脚本（本地 Node.js）

把 `english-vocabulary` 的原文件转换成统一格式 NDJSON/JSON：

- 输出 `books.json`
- 输出 `words.json`（去重）
- 输出 `book_words.json`

规范化建议：

- `word` 转小写、trim
- 去除重复 phrase/sentence
- 过滤异常空词条

## 6.2 分批导入云数据库

由于微信云数据库单次写入与请求体有大小限制，采用分批：

- `words`: 每批 200~500 条
- `book_words`: 每批 500~1000 条
- 失败批次可重试（记录断点）

## 6.3 导入后校验

至少校验：

- 每本词书 `book_words` 数量是否正确
- `book_words.word_id` 是否都能在 `words` 命中
- 随机抽样词条的翻译/例句字段完整性

---

## 7. 性能与成本建议（微信云开发）

- 高频查询尽量走索引字段（特别是 `next_review_at`）
- 单词详情可做本地缓存（小程序 storage）减少读次数
- `review_due_count` 可异步更新，避免每次实时 count
- 大词书建议懒加载，不要一次性下发完整词表

---

## 8. 安全与权限

- 用户数据只允许访问本人 `openid` 相关记录
- 管理导入函数需管理员鉴权
- 数据库权限建议：前端只读公开词库；学习写入全部通过云函数

---

## 9. 版本规划（建议）

- **V1（2~3 周）**：词库导入、学习/复习主流程、基础进度
- **V1.1**：复习算法优化（FSRS）、数据统计看板
- **V1.2**：错词本、收藏、多词书并行学习

## 10. 参数解释（给前后端对齐）

以下两个参数已按推荐定稿，补充说明如下：

1. **每次 session 默认值（已定 `10`）**
   - 含义：用户进入一次学习/复习流程时，默认先分配 10 个词
   - 作用：避免单次任务过长导致中途退出；同时允许用户在 `5~40` 之间自行调整

2. **评分档位（已定 `again/hard/good/easy`）**
   - 含义：用户每学完一个词后，选择记忆反馈档位
   - 用途：后端根据档位更新 SM2 参数，计算下次复习时间
   - 推荐映射：`again=忘记`，`hard=困难`，`good=记住`，`easy=很熟`

当前已无阻塞项，可以进入编码阶段。

## 11. 最小可用后端清单（MVP）

按优先级建议先做：

1. 集合：`books`、`words`、`book_words`、`user_word_state`、`user_book_progress`
2. 云函数：`getBookList`、`startSession`、`submitWordResult`、`getTodayPlan`
3. 导入脚本：`json-sentence -> 三张基础表`
4. 调度算法：SM2 简化版（已定）

这样可以最快跑通“选词书 -> 学习 N 个 -> 次日复习”的闭环。
